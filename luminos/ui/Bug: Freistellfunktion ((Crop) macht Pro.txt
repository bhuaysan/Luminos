● Now I have a complete picture. There are three distinct bugs:                                                        
                                                                                                                       
  1. Fixed zoom mode: _viewport_to_preview_norm computes (pos.x() + hval) / label_w — when the image is larger than the
   viewport, pos.x() tops out at the viewport edge, so you can only select the visible slice (e.g. 40% of the image at 
  150% zoom). The selection literally gets stuck.                                                                      
  2. Scroll wheel fires in crop mode: wheelEvent is not blocked in crop mode, so an accidental scroll changes          
  zoom/scroll offsets mid-drag, corrupting the start-point coordinates.                                                
  3. Start-point drift: The start viewport coordinate is re-converted to image-normalized on every drag event using the
   current hval/vval. If those change (via wheel scroll), the anchor point jumps.                                      
                                                            
  Fix strategy: Force fit-to-window when crop mode is activated (full image always visible), restore the previous zoom 
  when leaving crop mode. Additionally block wheel events in crop mode to prevent drift from (2) even in the fit-mode
  path.
